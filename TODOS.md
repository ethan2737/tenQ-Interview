# TODOS

## 1. 支持 PDF/docs 导入并切分单文件多题

- What: 支持导入 `PDF/docs`，并把一个文件中的多道面试题切分成独立题卡。
- Why: 这是当前第二类真实痛点资料来源，长文档里常有 20-30 题，只读原文很容易分神。
- Pros: 能把 Markdown MVP 的能力扩展到区块链资料，覆盖更完整的备面材料。
- Cons: 需要新增文档解析、题目边界识别、多题切分与对应测试，明显扩大当前范围。
- Context: 当前 MVP 已锁定只支持一题一文的 Markdown。PDF/docs 的核心难点不是“读取文件”，而是如何稳定识别题目边界，并确保切分后仍然“题问什么，答案就答什么”。
- Depends on / blocked by: 依赖 Markdown MVP 的导入、解析、抽段、生成、缓存链路先稳定。

## 2. 增加专题刷与随机刷复习模式

- What: 在基础题卡稳定后，增加 `专题刷` 和 `随机刷` 两种复习模式。
- Why: 这对应真实复习路径中的第二遍和第三遍，不是额外装饰功能。
- Pros: 能把当前的题卡生成器升级成真正可长期使用的复习工具，提升复用率和记忆效率。
- Cons: 需要新增题卡组织方式、抽样策略、更多前端状态和额外测试。
- Context: 当前 MVP 只保留按原始目录树顺序刷，这是正确的最小范围。专题刷和随机刷必须建立在题卡质量、缓存稳定和基础浏览体验可靠之后。
- Depends on / blocked by: 依赖 Markdown MVP 的数据层稳定，尤其是题卡质量与缓存正确性。

## 3. 生成模拟面试播客音频

- What: 基于题卡资产，支持按主题或侧重点自动组合 10 题，生成“面试官 + 面试者”的模拟面试播客音频。
- Why: 这是碎片时间复习的目标形态之一，能把坐在电脑前的背诵延伸到通勤、走路等场景。
- Pros: 扩展使用场景，提高题卡资产的复用率，形成更强的日常使用粘性。
- Cons: 需要新增组题逻辑、脚本编排、TTS/音频流水线、质量控制和更多状态管理，明显是下一阶段产品。
- Context: 当前已批准方案明确第一阶段只做 Markdown 文档压缩器。播客功能依赖题卡质量、题卡组织能力和稳定的数据层，否则只会更快传播低质量内容。
- Depends on / blocked by: 依赖题卡生成链路稳定，以及后续复习模式能力成熟。

## 4. 增加资料搜索能力

- What: 增加资料搜索能力，至少支持按文件名或题目快速定位题卡。
- Why: 当资料规模扩大到数百篇后，顺序刷之外会自然需要快速定位某道题。
- Pros: 提高后续复习效率，尤其是在主题复习、临时查漏或面试前快速定位某类题目时。
- Cons: 会引入额外状态、索引与交互设计，当前 MVP 不该分心。
- Context: 当前第一版已明确不做搜索，避免把产品从“复习台”做成“知识检索器”。但在 375 篇资料规模下，搜索迟早会成为合理能力。
- Depends on / blocked by: 依赖基础导入、目录树、题卡阅读和缓存链路稳定。
